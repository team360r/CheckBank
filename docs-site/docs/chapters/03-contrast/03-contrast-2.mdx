# Part 2: Fixing the Theme

In Part 1, your contrast checker caught two colours that fail WCAG AA:

- **`textSecondary`** (#B0B0B0) -- ratio 2.17:1 on white (needs 4.5:1)
- **`textDisabled`** (#D5D5D5) -- ratio 1.47:1 on white (needs 4.5:1)

Now you will fix them.

---

## The Fix

The goal is to darken each colour enough to meet its required threshold while keeping the visual hierarchy intact. Secondary text should still look secondary (lighter than primary text), and disabled text should still look muted.

  </TabItem>

```dart title="lib/theme/app_colors.dart (fixed)"

static const textSecondary = Color(0xFF757575);

// highlight-next-line
// FIX: Darkened to #9E9E9E — ratio ~3.5:1 on white (passes AA large text)
static const textDisabled = Color(0xFF9E9E9E);
```

  </TabItem>

### Why These Specific Values?

- **#757575** for `textSecondary` achieves approximately 4.6:1 on white, just clearing the 4.5:1 AA threshold for normal text. It is visibly lighter than `textPrimary` (#212121 at ~16.1:1), preserving the visual hierarchy.
- **#9E9E9E** for `textDisabled` achieves approximately 3.5:1 on white. This passes the 3.0:1 AA large text threshold. Disabled text is typically used at larger sizes or in non-critical contexts, so the large text threshold is appropriate. If your app uses disabled text at small sizes, you would need to push it darker still.

## Testing in Rendered Context

So far, you have tested colour constants directly. But in a real app, the colour that reaches the screen depends on the widget tree: themes, overrides, opacity, and background colours can all change the effective contrast.

A more thorough approach extracts the computed style from a rendered widget and checks the contrast against its actual background.

### The Concept

```dart title="Pseudocode — rendered contrast check"
// 1. Pump the screen
await tester.pumpWidget(MaterialApp(home: SettingsScreen()));

// 2. Find a specific Text widget
final textFinder = find.text('Account preferences');
final textWidget = tester.widget<Text>(textFinder);

// 3. Extract the effective text colour
final style = textWidget.style;
final textColor = style?.color ?? Theme.of(context).textTheme.bodyMedium!.color!;

// 4. Determine the background (scaffold surface colour)
final bgColor = AppColors.surface;

// 5. Compute and assert
final ratio = contrastRatio(textColor, bgColor);
expect(ratio, greaterThanOrEqualTo(kWcagAANormal));
```

## WCAG AAA -- The Enhanced Threshold

WCAG defines a second, stricter contrast level: **AAA (Enhanced)**, requiring **7.0:1** for normal text.

Not every colour needs to meet AAA. The standard itself acknowledges that AAA is not achievable for all text across an entire interface. But it is worth testing your most critical text -- headings, body copy, error messages -- against the enhanced threshold.

Let us check which CheckBank colours would pass AAA:

| Colour | Hex | Ratio on White | AA Normal (4.5) | AAA (7.0) |
|--------|-----|---------------|-----------------|-----------|
| `textPrimary` | #212121 | ~16.10:1 | PASS | **PASS** |
| `textSecondary` (fixed) | #757575 | ~4.6:1 | PASS | FAIL |
| `textDisabled` (fixed) | #9E9E9E | ~3.5:1 | FAIL (large only) | FAIL |
| `primary` | #00897B | ~4.32:1 | FAIL (large only) | FAIL |
| `error` | #D32F2F | ~5.76:1 | PASS (large) | FAIL |

Only `textPrimary` (#212121) passes AAA. The fixed `textSecondary` clears AA but falls short of 7:1. This is a common trade-off: secondary text that meets AAA would need to be so dark that it becomes indistinguishable from primary text, defeating the purpose of a visual hierarchy.

## Key Concepts

| Concept | What It Does |
|---------|-------------|
| Rendered contrast testing | Extracts computed colours from actual widget trees rather than testing constants in isolation |
| Visual hierarchy with contrast | Darkening colours to meet WCAG while preserving the distinction between primary, secondary, and disabled text |
| WCAG AAA (7.0:1) | Enhanced contrast threshold for critical content -- not required everywhere, but important for body text and long-form reading |
| Large text exception | Text >= 18pt (or >= 14pt bold) only needs 3.0:1, recognising that larger glyphs are inherently more legible |
